# ADR (Architecture Decision Record)

## 목적

의미 있는 architecture 결정과 그 배경, 대안, 결과를 기록합니다. 모든 변경을 ADR로 남기지 않고, 되돌리기 어렵거나 여러 기능에 영향을 주는 결정만 남깁니다.

## 작성 시점

- 새로운 핵심 의존성이나 인프라 연동을 도입할 때
- 데이터 모델의 핵심 구조(예: 회원과 소셜 계정 분리)를 결정할 때
- 인증/인가 방식을 결정할 때
- 기존 결정을 번복하거나 폐기할 때

## 작성 방법

1. [template.md](template.md)를 복사해 `docs/architecture/adr/NNNN-짧은-제목.md`로 저장합니다.
2. 번호(`NNNN`)는 4자리 일련번호를 사용합니다.
3. 결정 배경, 고려한 대안, 선택 이유, 영향과 후속 조치를 기록합니다.
4. 결정이 번복되면 기존 ADR을 삭제하지 않고 상태를 `Superseded`로 표시한 뒤 새 ADR을 작성합니다.

## 상태

현재 등록된 ADR이 없습니다. 첫 ADR은 architecture 결정이 실제로 확정되는 시점에 작성합니다.
